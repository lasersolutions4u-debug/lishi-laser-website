## 帖子类型

### 1. 问题-解决型帖子
**适用场景**: 引出痛点，介绍产品
**结构**: 行业常见问题 → 为什么发生 → 我们的解决方案 → CTA

### 2. 数字驱动型帖子
**适用场景**: 用数据吸引注意力
**结构**: 大数字开头 → 数据背景 → 数据含义 → CTA

### 3. 技术洞察型帖子
**适用场景**: 建立技术权威
**结构**: 行业技术观察 → 个人见解 → 开放讨论

### 4. 客户案例型帖子
**适用场景**: 社会证明
**结构**: 客户初始状态 → 使用后变化 → 具体数字 → 客户反馈

### 5. 行业趋势型帖子
**适用场景**: 思想领导力
**结构**: 市场趋势数据 → 趋势解读 → 对读者的影响 → 个人观点

## 发帖最佳实践

- 每帖 1 个核心观点
- 第一句抓住注意力（不要用"我很高兴宣布..."开头）
- 每 2-3 句换行
- 结尾问一个问题（促进评论互动）
- 配图优于无图，PDF优于配图
- 标签 3-5 个，放在末尾

### InMail 使用技巧

1. **Subject 是生死线** — 对方只看到前 40-50 字符。用具体数字开头，不用 "Introduction" 或 "Hello"
2. **第一句不要自我介绍** — 先给价值，再说你是谁
3. **全文 ≤150 词** — InMail 在手机上看，超过 3 屏就放弃
4. **一个明确 CTA** — "share the data" / "10-minute call" / "one question"
5. **不回复怎么办** — 7 天后发普通 Connection Request（不消耗 InMail 额度）
